# 地陪客户第一版验收清单

更新日期：2026-07-11

## 1. 用户端

- [x] 首页：品牌定位、服务边界、搜索、服务分类、服务能力示例、预约需求、入驻入口、帮助入口。
- [x] 城市选择：开放城市页和首页城市筛选。
- [x] 服务分类筛选：服务分类页、首页服务分类、精选服务者筛选。
- [x] 服务能力列表：展示不对应具体人员或真实交易的能力示例，并明确实际人员、资质、档期和价格由客服确认。
- [x] 服务能力详情页：展示服务类型、城市、参考价格、流程、范围、注意事项和预约入口；动态详情页不参与搜索收录。
- [x] 在线咨询按钮：联系客服、账号中心、订单进度、服务反馈、投诉举报等表单入口。
- [x] 预约需求提交：写入飞书“用户预约需求表”，并触发飞书机器人通知。
- [x] 登录 / 注册：第一版用账号中心提交注册、登录和订单列表协助线索。
- [x] 预约进度查询：使用 DP 预约编号和原手机号读取飞书脱敏状态；完整订单列表仍不公开。
- [x] 平台说明 / 帮助中心：已上线 `help.html`。

## 2. 服务者端

- [x] 服务者注册：首页入驻表单写入飞书“服务者入驻申请表”。
- [x] 入驻资料提交：包含城市、区域、服务类型、价格、时间、介绍、语言能力等字段。
- [x] 实名认证资料上传：第一版不在公开网页上传敏感证件，改为服务者中心提交资料补充线索，客服安全核验。
- [x] 服务项目发布：第一版由入驻资料和后台审核字段承接，审核后进入“服务者资料库”。
- [x] 服务价格设置：入驻表单与后台服务者资料库记录起步价和计费方式。
- [x] 订单查看：第一版由服务者中心提交状态咨询，客服在后台核验订单。
- [x] 接单 / 拒单：第一版由客服在“订单跟进表”和“客服跟进记录表”人工流转。
- [x] 服务状态管理：服务者中心可提交暂停、补充资料、状态咨询；后台维护可接单、忙碌、暂停、下架等状态。

## 3. 管理后台

- [x] 用户管理：飞书“用户预约需求表”和“客户咨询线索表”承接。
- [x] 服务者审核：飞书“服务者入驻申请表”承接。
- [x] 服务分类管理：飞书“城市与服务类目表”承接。
- [x] 城市 / 区域管理：飞书“城市与服务类目表”承接。
- [x] 服务者资料管理：飞书“服务者资料库”承接。
- [x] 订单管理：飞书“订单跟进表”承接。
- [x] 首页 Banner 管理：第一版使用静态首屏图和“平台配置表”记录公开配置。
- [x] 投诉 / 举报管理：飞书“投诉举报表”和公开投诉举报页承接。
- [x] 平台公告管理：第一版上线公开公告页 `announcements.html`，配置入口同步到“平台配置表”。

## 4. 运营链路

- [x] 飞书 10 张运营表已创建。
- [x] 预约、入驻、咨询、投诉、反馈、账号协助等表单已接入飞书接口。
- [x] Vercel 环境变量已配置。
- [x] 飞书机器人通知已接入。
- [x] 运营基础配置脚本已覆盖城市、规则、账号中心、订单进度、服务者中心、反馈、帮助中心和公告链接。
- [x] 域名 `www.dipeikehu.com` 已作为正式访问入口。

## 5. 上线质量与搜索准备

- [x] 8 个城市页已上线：7 个开放城市和 1 个重庆内测页。
- [x] 6 个服务分类页已上线：商务接待、城市陪同、旅游向导、办事协助、展会陪同、机场高铁接送。
- [x] 站点地图、robots、结构化数据、Open Graph、Web App 清单和自定义 404 已补齐。
- [x] API 已增加服务端字段长度、敏感词、蜜罐、日期、手机号、防重复和限流校验。
- [x] API 已强制 JSON、限制请求体、校验请求来源、协议同意、字段白名单、枚举值、日期范围、价格和人数。
- [x] 订单查询已改用飞书条件检索，不再按页扫描整张预约表。
- [x] 公开页面已完成 Chrome 1440/390/430 三种宽度共 102 组页面检查。
- [x] 深层 404、CSP、HSTS、无缓存敏感响应、XSS 输出编码和秘密泄漏检查已通过。
- [ ] Vercel WAF 持久化限流规则（首次启用需要在 Vercel 确认计费条款，代码层已保留双重限流）。
- [x] 预约编号使用不可预测的 16 位随机数字，进度查询需同时核验原手机号。
- [x] IndexNow 已准备自动提交脚本和站点密钥文件。
- [x] 静态审计、JavaScript 语法、接口校验、移动端和线上回归测试纳入发布流程。
- [ ] Google Search Console 所有权验证与站点地图提交（需站长本人账号生成专属令牌）。
- [ ] 百度搜索资源平台所有权验证与链接提交（需站长本人账号生成专属令牌）。
- [ ] 域名邮箱（待具备营业执照或选定可用的个人域名邮箱方案后开通）。

## 6. 第一版暂不做

- [ ] 真实账号登录系统。
- [ ] 在线支付、退款和发票。
- [ ] 服务者自行后台接单 / 拒单。
- [ ] 公开订单明细页。
- [ ] 身份证、合同、证照等敏感附件的网页上传。
- [ ] 即时聊天、地图定位、小程序和移动 App。

这些事项放入第二版或业务验证后再做，避免第一版过早变重。
