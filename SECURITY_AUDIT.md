# 地陪客户第一版安全与质量审计

审计日期：2026-07-11

## 已修复问题

1. 高风险：订单进度查询原先最多分页扫描 2000 条飞书记录，容易被重复请求放大。已改为飞书 `records/search` 条件检索，并限定返回字段与条数。
2. 高风险：表单 API 可绕过前端提交任意城市、服务类型、无效日期、价格和嵌套字段。已增加服务端字段白名单、类型、枚举、长度、日期、价格、人数和协议同意校验。
3. 中风险：API 未强制 `application/json`，也未检查请求体大小和跨站来源。已增加 415、413 和来源校验。
4. 中风险：客户端提交时间可被伪造。飞书记录现统一使用服务端上海时区时间。
5. 中风险：错误响应可能暴露飞书字段、记录 ID 或内部错误。公开响应现只返回用户编号和通用错误。
6. 中风险：动态订单结果使用 HTML 渲染。所有服务端字段均经过 HTML 实体编码，自动测试覆盖敏感字段不返回。
7. 中风险：深层 404 的相对 CSS、脚本和链接会解析到错误目录。现统一使用站点根路径。
8. 低风险：手机号曾出现在浏览器 `localStorage` 键名。现仅按表单类型记录提交时间，不保存联系方式。
9. 低风险：投诉内容前端声明 800 字但实际静默截断至 500 字。现按控件 `maxlength` 收集。
10. 低风险：首页 meta CSP 包含浏览器不支持的 `frame-ancestors`，会产生控制台错误。现仅通过 HTTP 响应头下发完整 CSP。
11. 高风险：接口曾信任任意 `*.vercel.app` 来源，攻击者可用自己的 Vercel 站点绕过来源白名单。现仅允许主域名、`www`、本项目固定 Vercel 域名和本地开发地址。
12. 低风险：静态资源曾允许最长一天的陈旧缓存，更新后可能短暂显示旧版样式或图片。现改为最多缓存五分钟并强制重新验证。

## 自动化验证

- 34 个 HTML 页面静态结构、链接、锚点、canonical、description、表单和 CSP 兼容性检查。
- 31 个 sitemap URL 与 14 个 SEO 落地页检查。
- Chrome 1440×900、390×844、430×932 共 102 组页面/视口检查。
- API 字段、协议、来源、请求体、敏感词、编号、状态隐私和限流单元测试。
- 使用真实飞书测试预约验证条件检索；错误手机号统一返回 404。
- Vercel 部署后运行全部公开 URL、嵌套 404、安全响应头和 API 防护检查。

## 剩余平台项

Vercel Serverless 内存限流只在单个运行实例内生效。代码已同时按 IP 和联系方式限制，并通过来源校验、蜜罐、字段白名单和请求体上限降低滥用风险。更强的跨实例限流应启用 Vercel WAF Rate Limiting；该功能首次启用需要账号所有者确认平台计费条款，因此未在本次代码提交中自动开启。
